## 1. 定位与目标

### 1.1 一句话定义

**会议实时助手**：用户在会议期间，系统持续监听对话音频，实时将对方的发言转译为文字，并从用户的记忆卡片库中自动匹配最相关的卡片，通过主界面和悬浮窗同时展示，帮助用户在不明显转移视线的前提下快速召回项目细节。

### 1.2 核心价值

- **会前（已有）**：素材 → 卡片提取 → 复习
- **会中（本功能）**：实时辅助 → 会议中即时提词 → 不被问住

### 1.3 成功指标

| 指标 | 目标 |
|------|------|
| 语音识别延迟 | 实时转字延迟 < 1.5 秒 |
| 匹配准确率 | 对方提问后 3 秒内，目标卡片出现在 Top 3 |
| 浮窗可用性 | 用户无需低头，视线偏移角 < 15° 即可读完卡片标题 |

## 2. 使用场景

### 场景 A：远程视频会议（主场景）

```
[对方] Zoom/Teams 语音 → [MacBook 系统音频] → [本 App 语音识别]
                                                      ↓
                                              实时转译 + 匹配卡片
                                                      ↓
                                         浮窗显示在摄像头正下方
                                    用户读浮窗时视线接近对着镜头
```

**关键约束**：用户面对摄像头坐下，浮窗必须在摄像头附近，否则失去"眼神接触"效果。

### 场景 B：线下会议

```
[对方] 面对面说话 → [MacBook 内置麦克风] → 同上流程
```

**差异**：环境噪音更大，识别准确率可能下降；浮窗位置更灵活，可放在桌面边角。

## 4. 功能规格

### 4.1 会议模式主界面

#### 顶部控制栏

| 元素 | 规格 |
|------|------|
| 开始/停止按钮 | 点击开始 → 持续监听 + 弹出浮窗；点击停止 → 关闭麦克风 + 关闭浮窗 + 保存记录 |
| 语言切换 | Toggle：`中文` / `英文`，切换时立即重启识别引擎（zh-CN / en-US）；默认 zh-CN |
| 状态指示 | 监听中：绿色脉冲点 + "正在监听…"；停止：灰色 |
| 清空按钮 | 停止状态下可用，清空当次转译记录（不删已保存内容） |

#### 左侧：实时转译面板

- **已识别文字**（黑色）：语音识别 `isFinal=true` 的结果，追加显示
- **识别中文字**（灰色斜体）：`isFinal=false` 的临时结果，实时刷新
- 文字区域可滚动，自动跟随最新内容滚动到底部
- 显示累计字数

#### 右侧：匹配卡片面板

- 实时展示 Top 4 匹配卡片，按相关度排序
- 每张卡片默认折叠，显示：类型标签 · 标题 · summary
- 点击任意卡片展开，显示完整 `details` 内容
- 同时只能展开一张（展开另一张时自动折叠当前）
- 无匹配时显示"继续说话…"提示，不显示空状态报错

---

### 4.2 摄像头悬浮窗（Camera Float Window）

#### 触发与生命周期

| 时机 | 行为 |
|------|------|
| 用户点击"开始会议" | 自动创建浮窗，定位到屏幕顶部摄像头正下方 |
| 有新匹配结果 | 浮窗内容实时更新（top 2-3 张卡片） |
| 用户点击"停止会议" | 自动关闭浮窗 |
| 用户手动关闭浮窗 | 浮窗消失，主界面会议继续运行；可点"重新显示浮窗"按钮找回 |

#### 浮窗视觉规格

```
┌─────────────────────────────────┐  ← 宽 340px，圆角 12px
│  ● 技术原理                     │  ← 类型标签（小字，带颜色）
│  TF-IDF 过滤噪声的原理          │  ← 卡片标题（中号，加粗）
│  我用 TF-IDF 过滤高频无意义词，  │  ← 关键句（小字，灰色）
│  让专业术语权重凸显。            │
├─────────────────────────────────┤  ← 分割线
│  ● 数据指标                     │
│  ESG 准确率 100%，Token 降 40%  │
│  我在 ESG 审计中降低了 40% API  │
│  调用成本，同时保持 100% 准确率。 │
└─────────────────────────────────┘
         ○ ● ○                      ← 分页点（表示共 3 张卡片）
```

| 属性 | 规格 |
|------|------|
| 尺寸 | 宽 340px，高自适应（每张卡约 90px，最多 3 张） |
| 默认位置 | 屏幕水平居中，距顶部 36px（MacBook Pro 摄像头缺口下方） |
| 层级 | `alwaysOnTop: true`，覆盖其他所有窗口（包括 Zoom） |
| 背景 | 白色，`opacity: 0.95`，轻微阴影 |
| 无边框 | `frame: false`，仅显示内容区域 |
| 可拖动 | 标题区域可拖拽，拖后位置持久化保存 |
| 卡片数量 | 显示 Top 2-3 张，底部分页点指示 |
| 切换方式 | 点击分页点 / 左右滑动切换卡片 |

#### 关键句提取规则

从卡片 `details` 字段提取**前两个完整句子**（以 `。！？.!?` 结尾，且每句 ≥ 15 字符）拼接展示。若 details 无完整句，取前 80 个字符 + `…`。

**示例：**
- details 原文：`我在2024年华威商学院创客马拉松中担任队长，带领6名来自不同国家的成员，在48小时极限压力下完成Debibi债务助手的开发与路演。通过BPMN流程图砍掉3个伪需求…`
- 关键句：`我在2024年华威商学院创客马拉松中担任队长，带领6名来自不同国家的成员，在48小时极限压力下完成Debibi债务助手的开发与路演。`

### 4.3 语音识别规格

| 参数 | 值 |
|------|----|
| 引擎 | macOS 原生 SFSpeechRecognizer（设备端 ASR）+ Web Speech API 兜底 |
| 模式 | `continuous: true`，`interimResults: true` |
| 语言切换 | `zh-CN`（中英混合）/ `en-US`（英文为主），切换时 abort + restart |
| 静音处理 | 识别引擎 `onend` 后自动重启（保持持续监听） |
| 错误处理 | `no-speech` 静默重启；其他错误显示提示，不中断会议 |

---

### 4.4 卡片匹配算法

#### 输入窗口

取转译记录**最后 300 字符** + 当前**识别中文字**，作为匹配查询文本。

> 只用最近内容而非全部记录，避免早期内容干扰当前匹配。

#### Tokenize 规则

```
英文：提取所有长度 ≥ 3 的字母/数字词（lowercase）
中文：提取所有 CJK 字符的单字 + 相邻双字（bi-gram）
```

#### 评分规则（v0.1 基线）

```
命中卡片 title / summary / tags 中的 token → +2 分
命中卡片 details 中的 token               → +1 分
最终得分 = 命中分之和（不做归一化）
```

> v0.1 用纯词面 BM25 风格评分。后续已升级为 FTS5 词法 + sqlite-vec 语义向量的混合检索（RRF 融合 + 置信度门控），见后续决策日志。

#### 输出

- 主界面：Top 4，得分为 0 的卡片不显示
- 浮窗：Top 3，得分为 0 的卡片不显示
- 两者独立更新，互不影响

### 4.5 会话自动保存

| 时机 | 行为 |
|------|------|
| 用户点击"停止会议" | 若转译记录 ≥ 50 字，自动保存 |
| 保存格式 | 以 `material:addText` 写入项目素材库 |
| sourceRef | `会议记录 · YYYY-MM-DD HH:mm` |
| 用户可见 | 保存后在素材 Tab 可见，可删除 |
| 保存失败 | 静默失败，不中断用户操作，console.error 记录 |

## 5. 技术架构要点

### 5.1 组件划分

```
renderer/pages/tabs/meeting-tab.tsx       主界面（现有，需扩展）
renderer/pages/meeting-float.tsx          浮窗渲染页（新建）
main/ipc/meeting.ts                        会议 IPC handler（新建）
```

### 5.2 浮窗 IPC 流

```
meeting-tab.tsx
  → ipcRenderer.invoke('meeting:open-float')   开始会议时
  → ipcRenderer.invoke('meeting:update-cards', cards[])  每次匹配更新
  → ipcRenderer.invoke('meeting:close-float')  停止会议时

main/ipc/meeting.ts
  → 创建/销毁 BrowserWindow（show:false → loadURL → show）
  → 转发 update-cards 到浮窗 webContents.send
  → 保存浮窗拖拽后的位置到 app_settings
```

### 5.3 浮窗 BrowserWindow 参数

```typescript
new BrowserWindow({
  width: 340,
  height: 'auto',   // 由内容决定，通过 IPC 动态调整
  alwaysOnTop: true,
  frame: false,
  transparent: false,
  resizable: false,
  movable: true,
  x: screenWidth / 2 - 170,
  y: 36,
  webPreferences: { contextIsolation: true, nodeIntegration: false }
})
```

## 6. 范围边界（Out of Scope）

| 功能 | 原因 |
|------|------|
| 本地离线语音识别（Whisper.cpp） | 集成复杂，后续考虑 |
| 会议录音保存（音频文件） | 隐私风险高，仅保存文字转译 |
| 多语言同时识别（zh+en 自动切换） | Web Speech API 不支持多语言并行，需手动 toggle |
| iOS / 手机端 | 后续独立 App |
| 提问历史回放 | 转译记录已保存到素材，可在素材 Tab 查看 |
| 实时打分 / 答案建议 | 超出辅助记忆范畴，会产生依赖 |
